# .claude/rules/ — プロジェクト固有規約の置き場

このディレクトリは、Claude Code の **native path-scoped rule 機構**（ファイル frontmatter の
`paths:` に一致する操作時のみ、そのルールファイルを自動ロードする仕組み）を使い、
プロジェクト固有の工学規約（テスト戦略・フロントエンド設計・Docker 運用など）を
必要な作業時にだけ読み込ませるための置き場です。

agent-forge のコア（`~/.claude/rules/`）はホスト非依存の基盤原則のみを持ち、
個人・チーム固有の工学的嗜好はコアから意図的に外されています
（`~/.claude/rules/README.md` の「空欄は意図的」参照）。ここに自分の規約を追加してください。

## `paths:` frontmatter の書き方（サンプル）

```markdown
---
# paths: に一致するファイルを操作するときだけ、Claude Code がこのルールを自動ロードする。
paths:
  - "**/*.test.ts"
  - "**/*.spec.ts"
  - "**/__tests__/**"
---

# テスト戦略ルール

ここに本文（原則・パターン・禁止事項など）を書く。
```

## 追加のしかた

1. このディレクトリに `<任意の名前>.md` を作成する（例: `testing.md` / `frontend.md` / `docker.md`）。
2. 先頭に上記サンプルのような `paths:` frontmatter を書き、対象ファイルパターンを指定する。
3. 本文に規約を書く。既存プロジェクトから移植する場合は原文をそのまま貼ってよい。

`paths:` を持たない全体適用の規約は、`.claude/rules/` ではなくプロジェクトルートの
`CLAUDE.md` に書くか、`~/.claude/rules/` 側（グローバル・全プロジェクト共通）に追加してください。
