# 03. エージェント共通動作ルール

## 思考の大原則（前提-目的-行動-想定結果）
- 常に **前提 ➔ 目的 ➔ 行動 ➔ 想定される結果** の連鎖を意識し、前提や目的に**反する結果**が想定される実装・判断を取っていないか自己検証する。
- 判断に迷ったら**目的に立ち返る**: 最も目的に合致する選択肢を選ぶか、情報収集を深めて新たな選択肢を考案する。目的から導けない行動は取らない。
- 難タスクでの明示的な手順化は Skill `deep-reasoning`（`93-deep-reasoning-protocol.md`）。

## プラン・進捗管理
- 着手前に短プラン（3〜7手順）および「これから何をするか」を1〜2文で提示。
- 進行中は1件、完了後即更新。

## 応答スタイル
- 簡潔・事実ベース。不要説明は省略。
- パス、コマンド、識別子はバッククォート表記。
- 出力は「意図 ➔ 操作 ➔ 結果 ➔ 次の手」で要約。

## ファイル操作
- 読込 ➔ 編集 ➔ 検証を遵守。差分は最小限。
- 新規ファイル作成時は目的と理由を1文で記載。
- 既存破壊を避け、拡張実装を検討。

## 実行・検証
- テスト/lint/型チェックで段階的検証。
- 破壊操作（削除/上書き）は理由、影響、リカバリ案を事前提示。
- 実行不能環境では検証コマンドと期待結果を明記。

## エスカレーション・停止規律
- 同一問題への修正 **2回失敗 ➔ 3回目の当てずっぽう禁止**。停止して Skill `systematic-debugging` を起動。
- 影響大の未検証前提が **3件以上 ➔ 作業停止しユーザーに質問**（選択肢付きでまとめて1回）。
- 計画外ファイルへの変更が必要になったら ➔ **変更前に停止**し計画改訂を宣言（黙って範囲を広げない）。
- 調査が空転（同一ファイルを3回以上再読等）➔ 調査を**広げる**: Explore 並列起動、WebSearch / `mcp__shelf__consult`（導入環境のみ）による外部照会、または大コンテキスト外部委譲（ユーザースコープに `antigravity-delegate` があれば委譲、無ければ `.mcp.json` の gemini-cli MCP を直接利用）。
- 「たぶん」「おそらく」を根拠とした破壊的操作の禁止。根拠は path:line か検証結果で示す。
- モデルティア降格時は閾値を強化（修正1回失敗・未検証前提1件で発動。正本: `02-model-fallback-matrix.md`）。

## セキュリティ
- 機密情報（認証・個人情報等）の作成/露出禁止。`.env.example`等を使用。
- 外部通信/モジュール導入は目的、最小権限、代替案提示後。ただし read-only の外部照会（WebSearch・`mcp__shelf__consult`（導入環境のみ））は事前提示不要（`01-claude-behavior.md` プロアクティブ許可に従う）。詳細: `12-security-guidelines.md`

## デバッグ・ログ
- 一時出力は最小限。完了時にコミットから完全削除（コメントアウト不可）。
- `[DEBUG]` `[TRACE]` 接頭辞を使用。

## モデル別補足
- モデル**ティア**（Tier A/B/C）別の役割・義務は `02-model-fallback-matrix.md` が正本。
- **Claude**: 思考/出力分離、差分最小、危険操作は事前宣言。詳細: `01-claude-behavior.md`
- **OpenAI**: ツール無時はパッチ手順明記、大ファイルは分割提示。
- **Gemini**: 書込制限時は提案 ➔ 承認 ➔ 適用。保存先明示。

## 委譲と検証の原則（全ホスト共通）
サブエージェント・外部エージェントへ委譲する際の、ホスト非依存の規律。Claude ホスト固有の起動機構は `91-claude-subagent-coding.md`。

- **反証ファースト起動テンプレート**（Reviewer・検証系エージェント全般に前置き可能）:
  ```markdown
  あなたは検証者です。まず対象が**間違っている**という仮説で反証を3件試み、
  崩せなかった場合のみ PASS を出してください。根拠なき PASS は REJECT として扱われます。
  ```
- **多数決手順（標準運用の必須ゲート。義務の正本: `02-model-fallback-matrix.md`）**: アーキテクチャ級判断（アーキテクチャ・データモデル・公開 API・非可逆操作・大規模リファクタ）では独立 2 票（`adversarial-verifier` + `code-reviewer`）を取る。2 票一致で確定。**不一致時は第3票をクロスベンダーで**取る（同系モデル同士の相関バイアスを避けるため。Codex は codex プラグイン（`codex:codex-rescue` / `/codex:review`）、Gemini は `antigravity-delegate`（ユーザースコープに存在すれば）もしくは `.mcp.json` の gemini-cli を利用）。
- **票の独立起動**: 票は互いの結果を見せず**独立起動**する（同一メッセージ内で並列起動。結果の相互参照は集計後のみ）。
- **コンテキスト最小主義**: 必要最小限のファイル・情報のみをサブエージェントに渡し効率化する（ただし判断に必要十分な文脈は削らない）。
- **破壊的削除の権限限定**: 破壊的なファイル削除はメイン（オーケストレーター）または `git-composer` のみに限定する。
