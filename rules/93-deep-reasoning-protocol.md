# 93. 深い推論プロトコル（Deep Reasoning）
難しい判断を暗黙の思考力に頼らず、明示的な手順で再現するための戦略。
**手続きの本体は Skill `deep-reasoning` に切り出してある**（92 と同じ「ルール=トリガ、スキル=手続き本体」方式）。
本ルールは「いつ発火し、どの優先度か」を定める薄い参照層。

## トリガと発火
以下のいずれかに該当したら、**必ず Skill `deep-reasoning` をロードして従う**（`/deep-reasoning`）。
- (a) Skill 記載の発火チェックリストに **1 項目以上**該当した時（= 非自明タスク全て。
  **全メイン共通の閾値**。正本: `02-model-fallback-matrix.md`）。
- (b) ユーザーが「深く考えて」「熟考して」「じっくり検討して」等と指示した時
  （**チェックリストによらず単独で即時発火**）。

## 要点（詳細は Skill 参照）
- 問題再述 ➔ 前提台帳（根拠・検証状態・影響度）➔ 仮説と**反証探索** ➔ 代替案 2 つ以上 ➔ プレモーテム ➔ コミット判定。
- 埋められない欄は捏造せず「未検証」と書き、影響大の未検証前提は検証するか質問する。
- 確信度「低」でのコミット禁止。`03-agent-behavior.md` のエスカレーション・停止規律へ。

## 92（plan mode）との関係
plan mode 中は `hosts/claude/92-plan-mode-protocol.md`（Skill `plan-deep-research`）が主プロトコル。
本スキルを二重実行せず、Step 2〜5（前提台帳・反証・代替案・プレモーテム）を
plan の設計判断部分に**組み込んで**適用する。
